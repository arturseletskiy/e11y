# Dead Letter Queue (DLQ) — Cucumber QA Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Verify that failed event deliveries are stored in the DLQ, that entries can be replayed through the pipeline, and that delete works — exposing that both `replay` and `delete` are non-functional stubs.

**Approach:** Configure E11y with a deliberately failing adapter plus DLQ enabled. Make HTTP requests via Rack::Test to generate events. Check DLQ storage state directly. For replay, verify the event reaches a secondary working adapter.

**Known bugs covered:**
- `DLQ::FileStorage#replay(id)` — the re-dispatch line (`E11y::Pipeline.dispatch`) is commented out. Method calls empty `increment_metric` and returns `true`. Events are NOT re-dispatched.
- `DLQ::FileStorage#delete(id)` — always returns `false`, implementation is a TODO.
- `DLQ::FileStorage#default_file_path` — calls `Rails.root.join(...)`, raises `NameError` when Rails is not loaded.

**Step 1: Write the feature file**

```gherkin
# features/dlq.feature
@dlq
Feature: Dead Letter Queue (DLQ) reliability

  # When an adapter fails to deliver an event, the DLQ captures it
  # so no data is permanently lost. Operators can later replay entries.
  #
  # BUG 1: DLQ::FileStorage#replay is a stub — events are NOT re-dispatched.
  # BUG 2: DLQ::FileStorage#delete always returns false.
  # BUG 3: Default file path calls Rails.root — fails without Rails.

  Background:
    Given the application is running
    And a failing adapter is configured with DLQ enabled

  Scenario: Failed delivery stores the event in the DLQ
    When I POST to "/orders" with params '{"order":{"order_id":"ord-1","user_id":"usr-1","items":"[]"}}'
    Then the DLQ should contain at least 1 entry
    And the DLQ entry should have an event_name field

  Scenario: DLQ entry contains the delivery error message
    When I POST to "/orders" with params '{"order":{"order_id":"ord-2","user_id":"usr-1","items":"[]"}}'
    Then the DLQ entry should have an error field

  @wip
  Scenario: Replaying a DLQ entry re-dispatches the event through the pipeline
    # BUG: replay calls empty increment_metric, returns true, does nothing else.
    # E11y::Pipeline.dispatch is commented out — doesn't exist.
    Given the DLQ contains 1 entry
    And a secondary working adapter is configured
    When I replay the DLQ entry
    Then the event should appear in the secondary adapter
    And the DLQ should be empty after replay

  @wip
  Scenario: Deleting a DLQ entry removes it permanently
    # BUG: delete always returns false regardless of input.
    Given the DLQ contains 1 entry
    When I delete the DLQ entry by ID
    Then the result should be true
    And the DLQ should be empty

  Scenario: DLQ with FileStorage persists entries to disk
    Given the DLQ uses file storage at a temporary path
    When I POST to "/orders" with params '{"order":{"order_id":"ord-3","user_id":"usr-1","items":"[]"}}'
    Then the DLQ file should exist on disk
    And the DLQ file should contain valid JSONL content

  @wip
  Scenario: DLQ FileStorage default path does not require Rails
    # BUG: default_file_path calls Rails.root.join("log", "e11y_dlq.jsonl")
    # Raises NameError: uninitialized constant Rails in non-Rails contexts.
    When I create a FileStorage without an explicit path
    Then no NameError should be raised
```

## Task 2: Step definitions

**Files:**
- Create: `features/step_definitions/dlq_steps.rb`

**Step 1: Write step definitions**

```ruby
# features/step_definitions/dlq_steps.rb
# frozen_string_literal: true

require "tempfile"

# A simple adapter that always raises on write.
FailingAdapter = Class.new(E11y::Adapters::Base) do
  def write(_event_data)
    raise StandardError, "Simulated delivery failure"
  end
  alias deliver write
end unless defined?(FailingAdapter)

Given("a failing adapter is configured with DLQ enabled") do
  @failing_adapter = FailingAdapter.new
  E11y.configuration.adapters[:default] = @failing_adapter
  E11y.configuration.dlq.enabled = true
  clear_events!
end

Then("the DLQ should contain at least {int} entr(y|ies)") do |min|
  dlq = E11y.configuration.dlq
  size = dlq.storage.size rescue dlq.size rescue dlq.all_entries.size
  expect(size).to be >= min,
    "Expected DLQ to contain >= #{min} entries, got #{size}."
end

Then("the DLQ entry should have an event_name field") do
  dlq = E11y.configuration.dlq
  entry = (dlq.storage.all_entries rescue dlq.all_entries).first
  expect(entry).not_to be_nil, "DLQ is empty — no entries found"
  has_name = entry.key?(:event_name) || entry.key?("event_name")
  expect(has_name).to be(true),
    "DLQ entry missing :event_name. Keys: #{entry.keys.inspect}"
end

Then("the DLQ entry should have an error field") do
  dlq = E11y.configuration.dlq
  entry = (dlq.storage.all_entries rescue dlq.all_entries).first
  has_error = entry&.key?(:error) || entry&.key?("error") ||
              entry&.key?(:error_message) || entry&.key?("error_message")
  expect(has_error).to be(true),
    "DLQ entry missing error info. Keys: #{entry&.keys.inspect}"
end

Given("the DLQ contains {int} entr(y|ies)") do |count|
  post "/orders", params: '{"order":{"order_id":"ord-replay","user_id":"usr-1","items":"[]"}}',
       "CONTENT_TYPE" => "application/json"
  dlq = E11y.configuration.dlq
  size = dlq.storage.size rescue dlq.all_entries.size
  expect(size).to be >= count
  @dlq_entry_id = (dlq.storage.all_entries rescue dlq.all_entries).first&.dig(:id) ||
                  (dlq.storage.all_entries rescue dlq.all_entries).first&.dig("id")
end

Given("a secondary working adapter is configured") do
  @secondary_adapter = E11y::Adapters::InMemory.new
  E11y.configuration.adapters[:secondary] = @secondary_adapter
end

When("I replay the DLQ entry") do
  dlq = E11y.configuration.dlq
  @replay_result = dlq.replay(@dlq_entry_id)
end

Then("the event should appear in the secondary adapter") do
  expect(@secondary_adapter.event_count).to be >= 1,
    "Expected replayed event in secondary adapter, got 0. " \
    "BUG: DLQ#replay is a stub — E11y::Pipeline.dispatch is commented out."
end

Then("the DLQ should be empty after replay") do
  dlq = E11y.configuration.dlq
  size = dlq.storage.size rescue dlq.all_entries.size
  expect(size).to eq(0),
    "Expected DLQ empty after replay, but still has #{size} entries."
end

When("I delete the DLQ entry by ID") do
  dlq = E11y.configuration.dlq
  @delete_result = dlq.delete(@dlq_entry_id)
end

Then("the result should be true") do
  expect(@delete_result).to be(true),
    "Expected delete to return true, got #{@delete_result.inspect}. " \
    "BUG: DLQ#delete always returns false (unimplemented TODO)."
end

Then("the DLQ should be empty") do
  dlq = E11y.configuration.dlq
  size = dlq.storage.size rescue dlq.all_entries.size
  expect(size).to eq(0)
end

Given("the DLQ uses file storage at a temporary path") do
  @dlq_temp = Tempfile.new(["e11y_dlq_test", ".jsonl"])
  E11y.configuration.dlq.storage = E11y::Reliability::DLQ::FileStorage.new(
    file_path: @dlq_temp.path
  )
  E11y.configuration.dlq.enabled = true
end

Then("the DLQ file should exist on disk") do
  expect(File.exist?(@dlq_temp.path)).to be(true),
    "DLQ file not found at #{@dlq_temp.path}"
end

Then("the DLQ file should contain valid JSONL content") do
  lines = File.readlines(@dlq_temp.path).map(&:strip).reject(&:empty?)
  expect(lines.size).to be >= 1, "DLQ file is empty"
  lines.each_with_index do |line, i|
    parsed = JSON.parse(line) rescue nil
    expect(parsed).not_to be_nil,
      "Line #{i + 1} is not valid JSON: #{line.inspect}"
  end
ensure
  @dlq_temp&.close
  @dlq_temp&.unlink
end

When("I create a FileStorage without an explicit path") do
  @dlq_init_error = nil
  begin
    E11y::Reliability::DLQ::FileStorage.new
  rescue NameError => e
    @dlq_init_error = e
  end
end

Then("no NameError should be raised") do
  expect(@dlq_init_error).to be_nil,
    "Got NameError: #{@dlq_init_error&.message}. " \
    "BUG: DLQ::FileStorage#default_file_path calls Rails.root — requires Rails."
end
```
